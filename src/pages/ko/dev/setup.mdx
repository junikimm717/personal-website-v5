# 개발환경

## 터미널

컴퓨터를 상당히 많이 갖고 있으며, 무슨 컴퓨터든 환경을 비슷하게 설정하려고
노력하고있습니다. 터미널에서 효율하게 탐색하기 위해 tmux 랑 oh-my-zsh를 주로
사용합니다. tmux는 제가 직접 개발한
[프로그램과](https://github.com/junikimm717/tmuxs) 같이 사용하며, 제가
이동하고 싶은 프로젝트로 즉시 갈수 있습니다. 가급적이면
아래처럼 한 키로 다이렉토리를 검색할 수 있도록 설정 합니다:

```bash
export WORKSPACES=$HOME/Documents:$HOME/docs/programming:$WORKSPACES
bindkey -s "^f" "tmuxs -d 4\n"
```

## 편집기

9학년때 빔 (Vim) 이라는 편집기를 발견한후, 여전히 사용하고 있었습니다.
현제는 LSP 와 색상 테마를 바꿀 수 있는 기능을 구현했으므로, 저만의 IDE 라고
볼수도 있습니다.
Docker 이미지가 커밋마다 빌드가 되며, 본인이 즉시 사용할수 있습니다.

서버 아니면 컨태인어 환경에서 급하게 편집기를 설정해야될 경우에는,
제할수 없이 가 구현한 [빔 설정
파일을](https://git.junickim.me/junikimm717/vim2023) 저장해서 제가 편집기에서 가장 좋아하는 기능들을 즉시
사용할 수 있습니다. 아래처럼 터미널에다가 복사해도 됩니다.

```bash
# 모든 기능이 있는 설정 파일
curl -LJ https://git.junickim.me/junikimm717/vim2023/raw/branch/master/lsp.vimrc -o $HOME/.vimrc
# 플러그인을 최소한 포함하는 설정 파일 (Git 은 요구한다)
curl -LJ https://git.junickim.me/junikimm717/vim2023/raw/branch/master/minimal.vimrc -o $HOME/.vimrc
# 플러그인이 전혀 안 포함되있는 설정 파일
curl -LJ https://git.junickim.me/junikimm717/vim2023/raw/branch/master/noplug.vimrc -o $HOME/.vimrc
```

## 창 관리자

MacOS 나 Linux 를 사용하고 있든, 최대한 비슷하게 설정합니다. 다만, Linux 에서
사용하는 MOD 키를 MacOS에서는 Alt 으로 대체했습니다.

- `mod-shift-a` 터미널을 열기
- `mod-shift-return` Tmux가 실행하고있는 터미널을 열기
- `mod-shift-w` 웹 브라우저를 열기
- `mod-space` 어플 선택 메뉴를 열기 (Raycast 아니면 Rofi)

어플은 제가 직접 세운 제도를 따라서 창으로 이동합니다.

1. 터미널
2. 웹 브라우저
3. 다른 편집기/개발 도구
4. 줌/회의 소프트웨어
5. 가상 머신 소프트웨어
6. 음악 플래이어
7. 컴퓨터 설정 도구
8. 디스코드
